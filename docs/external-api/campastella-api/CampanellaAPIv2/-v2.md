# /v2

サービスのメインのバージョンです。機能改修などで後方互換がなくなった場合、/v3 などの新しいバージョンとして実装される可能性があります。  
(サポートするのは最新のバージョンのみです。例えば`/v3/test`がリリースされた場合、`/v2/test`は Deprecated になります)

[[/v1]]にあった構成上の問題や、セキュリティ上の問題に対応するために更新されました。お手数おかけしますが[[/v2]]への移行をお願いします。

## 認証

リクエストに際して Authorization ヘッダによる Bearer 認証が必要です。
認証トークンを生成し、以下のような内容を HTTP リクエストヘッダにセットしてください。

```
Authorization: Bearer <token>
```

クライアントの関係上 Authorization ヘッダをセットできない場合は、エンドポイントのアクセス時に access_token クエリパラメータを用いて認証トークンをセットしてください。
**ただしこの方法は非推奨であり、可能な限りは ↑ の方法を行なってください。**

```
https://api.campanella.jp/v2/staff?access_token=<token>
```

### GAS のサンプルコード

以下のように UrlFetchApp.fetch()の options について、headers キーを用いて設定してください。

```
let options = {
  method: 'post',
  headers: {
    'Authorization': 'Bearer <token>', // これが必要
  },
  contentType: 'application/json',
  payload: JSON.stringify(payload),
  muteHttpExceptions: true,
}

try {
  var response = UrlFetchApp.fetch(API_URL, options);
  Logger.log(response.getContentText());
} catch (error) {
  Logger.log(error.toString());
}
```
