# Campanella API v3

## はじめに

メンテナンス性と拡張性を重視した改良版 API です。  
基本的な機能は v2 と同じですが、ログ周りが強化されたなど機能強化が行われています。  
V2 は機能改修は行われませんが、不具合修正などのメンテナンスは今後も行われます。

## 主な変更点

- ログ強化
  - API のアクセスログが記録されるようになり、より多くの情報が記録されるようになります。
- DELETE 機能の強化
  - force=true パラメータを付けることで、外部キー制約に起因するデータをスマートに削除できるようになります。
- 画像変換機能

## V2 からの破壊的変更

- API のエンドポイントの名称が変更されます。機能の塊をわかりやすくするための変更です。
  - 例：`/v2/staff` は `/v3/cms/staff` に変更されます。
- GET リクエストについて `activated_only` パラメータによるリクエストが廃止されます。
- POST 系リクエストでは、生成後の ID を含むエンドポイントを Location ヘッダで返却します。
- VRCID において追加・更新用エンドポイントが POST から PUT に変更されます。また以前は新規追加時は `201 CREATED` 、更新時は `200 OK` を返却していましたが、一律で `200 OK` を返却するようになります。
- VRCID の操作エンドポイントについて、 `display_name` パラメータの代わりに `vrc_display_name` を使用するようになります。
- 日付情報をエンドポイントで渡す際、UNIX タイムスタンプでデータを渡すと以前は UTC として解釈されていましたが、JST として解釈されるようになります。
- VRCDisplayMessage において key は少なくとも 1 文字以上が必要となります。また、英数字・アンダーバー・ハイフン・ピリオドのみの文字種で構成される必要があります。
  - 新しく追加された VRCDisplayImage にも同様の制約があります。
  - この制限は APIv2 には後方互換の観点でありませんが、v3 で利用できなくなるので移行することを推奨します。
- 一部の DB 挿入時・更新時のエラーが API 側でハンドリングされるようになりました(主に文字数・文字種制限など)。それに伴ってこれまで 500 エラーだった処理は一部 400 エラーとなります。詳細は下記の変更内容を参照してください。
- その他一部のエンドポイントの返却する情報に変更があります。
