# 外部 API 仕様書の配置場所

Campastella Web では、外部 API（キャストや画像データなど）の仕様書をこのディレクトリにまとめて保管します。  
提供いただいたドキュメントを `docs/external-api/<任意の名前>.md` として追加してください。

## 運用ルール

1. ファイル名にはバージョンや日付を含める（例: `events-v1-2024.md`）。
2. API レスポンスの JSON サンプルや注意事項もこのディレクトリに追記。
3. 変更があった場合は Pull Request で差分を確認できるようにし、利用側のコード（`src/lib/external.ts` など）も同時に更新。

## 参照方法

- 開発メンバーは、このフォルダのファイルを参照しつつ型定義やフェッチロジックを実装します。
- `src/lib/external.ts` に定義している Zod スキーマと仕様がずれた場合は、まず仕様書を更新 → コードを修正 → `pnpm typecheck` / `pnpm lint` を実行して整合性を確認してください。
- Campanella API については `campastella-api/CampanellaAPIv3/README.md` を最新版の一次資料とし、`CampanellaAPIv2/` は参照のみとします。

仕様書の原本が別システムで管理されている場合でも、ここには最新コピーを置くようにしておくとレビューやオンボーディングがスムーズになります。
